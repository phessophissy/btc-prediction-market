#  Reference — Section 37

## Overview

This document covers the  subsystem of the BTC Prediction Market platform,
focusing on operational patterns and integration points.

## Architecture

The  module interacts with:
- Smart contract layer (Clarity on Stacks)
- SDK service abstraction
- Frontend presentation components
- CLI tooling for operators

## Configuration

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `enabled` | boolean | `true` | Enable/disable the  subsystem |
| `threshold` | number | `37` | Threshold for triggering  actions |

```bash
NEXT_PUBLIC__ENABLED=true

```

## API Reference

### `getStatus(marketId: number)`

Returns the current  status for a specific market.

**Parameters:**
- `marketId` (number) — The on-chain market identifier

**Returns:** `Promise<Status>`

**Example:**
```typescript
const status = await getStatus(1);
console.log(status.phase); // 'active' | 'pending' | 'completed'
```

## Error Handling

### Common Error Codes

| Code | Constant | Description | Resolution |
|------|----------|-------------|------------|
| u1001 | ERR-MARKET-NOT-FOUND | Market ID does not exist | Verify market ID via `get-market-count` |
| u1002 | ERR-MARKET-CLOSED | Market past settlement height | Cannot bet on closed markets |
| u1007 | ERR-BET-TOO-SMALL | Below MIN-BET-AMOUNT (10,000 µSTX) | Increase bet to at least 0.01 STX |
| u1012 | ERR-NOT-READY-TO-SETTLE | Settlement height + 6 not reached | Wait for block maturation |

### Error Recovery

When encountering transient errors (network timeouts, API rate limits):
1. Implement exponential backoff starting at 2 seconds
2. Maximum 3 retry attempts
3. Log the failure for monitoring
4. Surface user-friendly error messages
