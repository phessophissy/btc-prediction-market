# Docs Index

This folder breaks the project into smaller references instead of forcing every detail into the root `README.md`.

## Sections

- `architecture/`: how the repo is organized and how data moves
- `contracts/`: contract versions, fee model, admin controls, and error codes
- `frontend/`: UI structure, wallet flow, and environment notes
- `sdk/`: TypeScript SDK surface area
- `cli/`: command-line tooling status
- `deployment/`: mainnet deployment notes and release checklists
- `development/`: local workflows and troubleshooting
- `reference/`: quick lookup tables for contracts, fields, fees, and outcomes
- `security/`: assumptions and operational safety notes
- `testing/`: current strategy and gaps
- `product/`: roadmap, user journeys, and known gaps
